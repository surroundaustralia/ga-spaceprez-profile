Abstract

This is a data profile defined to capture the data requirements of the SpacePrez tool.

SpacePrez is the spatially-configured Prez application.

Note that most of the constraints that this profile places on data are inherited from the OGC LDA PI Profile which ensures RDF spatial data is formulated according to GeoSPARQL 1.1 and that it has all the elements needed for OGC API - Features Core APIs.

This specification is not to be used for testing conformance of RDF resources to this profile. That role belongs to the validation resource within this profile:

  • https://w3id.org/profile/spaceprez/validation

For the list of all resources within this profile, see the profile definition:

  • https://w3id.org/profile/spaceprez

1. Introduction

SpacePrez is a spatial data configured version of the Prez application. Prez is a spatial data, Linked Data API, that is, an application used to deliver spatial data in Semantic Webref form, over the web.

In addition to providing data according to Linked Data and Semantic Web conventions, SpacePrez is structured according to the OGC API Featuresref standard.

Prez and SpacePrez draw their data from RDF graph databases which must contain data constructed according to the GeoSPARQL 1.1 standardref. GeoSPARQL provides a specification of requirements for data and a validator which can be used to validate that data does, in fact, conform to GeoSPARQL 1.1.

The OGC LD API Profileref is a profile - a supplementary constraining specification - of GeoSPARQL 1.1 that provides requirements for OGC APIs based on GeoSPARQL data. The OGC LD API Profile also supplies a validator that can be used to validate data according to its rules.

The document is the specification of requirements for the SpacePrez Profile. This is a profile of the OGC LD API Profile and contains just a few additional rules for data, on top of those from GeoSPARQL 1.1 and the OGC LD API Profile, that ensure that the system-specific data requirements of the SpacePrez tool are met. Like the GeoSPARQL and OGC LD API Profile, this profile also provides a validator that can be used to validate data for SpacePrez.

Since this profile only really makes sense within used with the OGC LD API and GeoSPARQL 1.1, this profile also contains a "compounded validated" which contains this profile, the OGC LD API and GeoSPARQL 1.1. validators all together. It can be used for complete data validation for SpacePrez.

This specification's rules/requirements - are given IDs and are indicated in red text in the next section.

2. Requirements

SpacePrez, the tool, can only interpret GeoSPARQL 1.1ref data with GeoJSON geometry serializations, so:

Requirement GJ: All GeoSPARQL Geometry objects must contain a GeoJSON serialization

Note that since much GeoSPARQL data is presented with WKT geometry serializations, this profile provides a Python script that can be used to convert those serializations to GeoJSON.
